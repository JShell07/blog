WebRTC 如何避免拥塞？

Posted on 2018-06-03 | Edited on 2018-07-17 | In WebRTC

WebRTC 如何避免拥塞？

1. WebRTC 网络传输背景

1.1. WebRTC && 多数直播

大多数视频直播平台

子项 说明
传输协议 RTMP（Real Time MEssaging Protocol)
优点 使用RTMP， 其基于TCP传输，跟flash等流媒体服务支持比较好，同时CDN支持良好
缺点 直播音视频数据量大，实时性要求比较高，TCP的重传机制和拥塞机制不适用于实时传输。传输控制依赖于TCP本身协议控制机制，网络成本较大不够灵活。

音视频对网络丢包有一定程度天然容忍性（常见优酷，爱奇艺都有缓冲）。使用UDP传输是不错到选择。

Docker

1. Docker 背景

1.1. Docker 是什么

Docker 是 PaaS 提供商 dotCloud 开源的一个基于 LXC (LXC 其并不是一套硬件虚拟化方法 无法归属到全虚拟化、部分虚拟化和半虚拟化中的任意一个，而是一个操作系统级虚拟化方)的高级容器引擎，源代码托管在 Github 上, 基于go语言并遵从Apache2.0协议开源。

Docker 属于 Linux 容器的一种封装，提供简单易用的容器使用接口。**它是目前最流行的 Linux 容器解决方案。
